C++编程技术

C++ VS C

  1. C++声明的位置没有限制，不像C一定要在使用之前

  2. C++允许直接使用结构体名定义实体，即不需要再用struct node定义实体，如：node a;

  3. C++有了运算符new和delete来完成动态内存分配

  4. C++增加了引用(references)类型，引用是让变量i拥有别名j，这样i和j都代表同一块地址中的内容。

  5. C++对const语义进行了扩展

    • 一般常量

      对于一般常量来说，C++中用const声明的常量在后续的编译过程中就可以直接访问，而C需要在程序执行的时候才可以访问。

    • 常指针和常引用

      对于指针和引用const的位置不同含义不同

    • 常成员函数和常数据成员

      使用const关键字说明的成员函数，只有常成员函数能操作常量和常对象

      待学了面向对象后补充

  6. C++指针的声明类型必须和指向的实体类型一致，不像C一样有缺省的自动转换，必须强制转换

  7. C++中int和char不再等价，sizeof('A')是1而sizeof(65)是2

  8. C++中的struct可以在内部定义成员函数，如果定义成员函数此struct的定义必须作为全局对象。

    struct是class的特例

C++存储技术

  1. C++存储类型

    • extern：全局变量需要在外部文件引用时，外部文件会加extern
    • static：局部的稳定变量，和全局变量一样在永久存储区中，只在作用域中生效
    • register：用于修饰int和char，表示将变量值保存于寄存器中

  2. C++存取修饰符

    • const：常数型
    • volatile：暂态型，用于说明该变量有可能被没有显式说明的方式改变，防止编译器自动优化

  3. C++对象的生存期

    • 局部对象：对象被定义时调用构造函数，程序退出定义对象所在的程序块时，调用析构函数。作用域在程序块内。
    • 静态对象：程序第一次执行定义的静态对象->程序结束。作用域在某个文件内。
    • 全局变量：程序开始时，对象被创建->程序结束时，对象被析构。作用于在包含该文件的程序内。

    类域：类的作用域，类定义中花括号括起的部分。类域中定义的变量只能用static修饰符，定义的函数也不能用extern。某个类A中成员M在以下情况下具有类A的作用域：

    • M出现在类A的某个成员函数中，且该函数没有定义同名标识符（？）
    • 类A的对象a取用到M，a.M
    • 类A的对象指针pa取用到M，pa->M
    • 使用作用域运算符所限定的该成员中，A::M（？）

C++标准库技术

C++标准库增进了I/O流技术，增加了String对象，各种容器、算法等，这些都放在命名空间std中，而旧C++的头文件如：<iostream.h>不包含std。为了兼容，将旧的C++头文件包装后去掉.h作为包含std的头文件，如：<iostream>，C的头文件去掉.h的同时前面加上c如<cstdio>
